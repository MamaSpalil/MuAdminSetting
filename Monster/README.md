# Monster assets

Папка содержит 3D-модели монстров MU Online (`Monster<NN>.bmd`).
Используется `MuAdmin.Core.Assets.AssetsManager` для определения
наличия модели у монстра по индексу — это видно в боковой панели
редактора `MonsterSetBase*.txt` (поле «Модель: BMD есть / нет .bmd»).

## Имена файлов

- `Monster<NN>.bmd` — 3D-меш клиента (`NN` может быть как
  двух-, так и трёхзначным: `Monster01.bmd`, `Monster25.bmd`,
  `Monster123.bmd`). Бинарный формат Webzen.
- (опционально) `Monster<NN>.{png,jpg,jpeg,bmp}` — растровый рендер,
  если хотите видеть в гриде каталога монстров (`Monster.txt`)
  настоящую картинку, а не плейсхолдер `M:N`. Менеджер ассетов
  подхватит файл автоматически.

## Почему BMD не отрисовывается напрямую

`*.bmd` — это формат 3D-модели Webzen, а не растровое изображение.
Декодировать его без 3D-движка (DirectX / OpenGL) нереалистично,
поэтому MuAdmin отображает плейсхолдер с индексом монстра, но
помечает: «модель есть» / «нет .bmd».

## Легальность

Файлы Webzen не должны добавляться в публичные репозитории. Кладите
сюда модели только из собственной поставки клиента.
